✅ Replit VidLab Auto Multi-Chunk + Progress Bar Prompt

Task:

Enhance my existing VidLab Flask app (Python + FFmpeg + BLIP-2) to process videos longer than 5 minutes safely on Replit free-tier and display real-time progress to the frontend.

1️⃣ Auto-Split Video
Check uploaded video duration (ffprobe)
If duration > 5 min (300s):

Split into sequential 5-minute chunks using FFmpeg stream copy:

ffmpeg -i input.mp4 -ss <start> -t 300 -c copy chunk_<n>.mp4
Each chunk saved in /tmp with unique UUID filenames.
Delete chunk after processing to save disk space.
2️⃣ Frame Extraction & BLIP-2
Extract frames for each chunk:
Use safe FPS for Replit: 1–2 fps or native frames (-vsync 0)
Limit max frames per chunk = 100
Delete frame folder after prompt generation to save disk
Run BLIP-2 (or lighter variant if needed) per frame to generate scene captions
Aggregate consecutive frames → scene-level prompts
3️⃣ Real-Time Progress Tracking
Maintain global chunk_progress dictionary in backend keyed by video_id
Track:
current_chunk, total_chunks
frame_progress per chunk
status string (Processing chunk X of Y)
Provide new API route /progress/<video_id> that returns current chunk & frame progress as JSON
4️⃣ Frontend Polling
Add JS to poll /progress/<video_id> every 0.5–1 sec
Update progress bar width and text dynamically:
Show: Chunk X of Y, frame N
When processing complete → show Processing complete!
5️⃣ Free-Tier Safety
Sequential chunk processing → prevent CPU overload
Cleanup /tmp before and after each chunk
Limit frames per chunk (max 100) → protect memory / disk
Handle FFmpeg timeout & disk quota errors → return JSON error messages
6️⃣ Aggregated JSON Output
Merge all chunks into single JSON for frontend:
{
  "scenes": { ... merged scene prompts ... },
  "chunks": [
    { "chunk_id": 1, "start_sec": 0, "end_sec": 300, "scene_count": 5, "frame_count": 100 },
    { "chunk_id": 2, "start_sec": 300, "end_sec": 600, "scene_count": 4, "frame_count": 92 }
  ],
  "total_scenes": 9,
  "total_frames": 192,
  "total_chunks": 2,
  "is_multi_chunk": true,
  "video_duration": 583.2
}
7️⃣ Deliverables
/analyze-scenes Flask route updated to:
Auto-split long videos
Sequentially process chunks with safe frame extraction
Generate BLIP-2 scene prompts
Track chunk_progress per chunk/frame
Merge results into final JSON
/progress/<video_id> route for real-time frontend updates
Frontend JS snippet to poll progress and update progress bar + text
Automatic cleanup of /tmp after each chunk
Safe for Replit free-tier, no external APIs, max frames limited

Bottom-line:

Any uploaded video >5 min → backend auto-splits → chunk-by-chunk frame extraction + BLIP-2 captions → real-time progress updates → aggregated JSON → fully safe for Replit free-tier.
